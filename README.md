## 📌 Описание задания
Реализовать систему обмена сообщениями с использованием брокера RabbitMQ. Настроить взаимодействие между веб-интерфейсом (Producer) и фоновым обработчиком (Worker) внутри Docker-контейнеров.
Результат доступен по адресу [http://localhost:8080](http://localhost:8080).

---

## ⚙️ Как запустить проект

1. Клонировать репозиторий:
   ```bash
   git clone <ссылка на репозиторий>
   cd nginx-lab
Запустить контейнеры:
```bash
docker-compose up -d --build
```
Запустить воркер (обработчик) внутри контейнера:
```bash
docker exec -it lab7_php php worker.php

Открыть в браузере:
```http://localhost:8080```

📂 Содержимое проекта

* docker-compose.yml — конфигурация сервисов PHP, Nginx и RabbitMQ.
* www/send.php — веб-интерфейс (Producer) для отправки сообщений.
* www/worker.php — консольный скрипт (Consumer) для приема сообщений.
* www/QueueManager.php — вспомогательный класс для управления очередью.
* www/vendor/php-amqplib/ — библиотека для работы с AMQP (установлена вручную).

🛠 Реализация и решение проблем

-   В ходе работы возникли сложности с автоматической установкой зависимостей через Composer из-за сетевых ограничений. Проблема была решена следующим образом:
1.  Локальная настройка PHP: Установлен интерпретатор PHP 8.x с включенными расширениями openssl и curl.
2.  Manual Install: Библиотека php-amqplib скачана напрямую из репозитория и размещена в папке vendor.
3.  Custom Autoloader: Реализован собственный загрузчик классов на базе spl_autoload_register для корректной работы библиотеки без autoload.php от Composer.

✅ Результат
- Система обмена сообщениями успешно настроена. PHP-контейнер стабильно взаимодействует с RabbitMQ, обеспечивая асинхронную передачу данных.
